Módulo 01

Fundamentos Teóricos

🌳

¿Qué es un Árbol?

Un árbol es una estructura de datos jerárquica compuesta por nodos. Cada árbol tiene un nodo raíz del que se derivan hijos, formando una relación padre-hijo sin ciclos.

Propiedades clave:
• Un solo nodo raíz
• Cada nodo tiene un padre (excepto la raíz)
• Sin ciclos (acíclico)
🔍

Árbol Binario de Búsqueda (BST)

En un BST, para cualquier nodo N: todos los valores del subárbol izquierdo son menores que N, y todos los del derecho son mayores.

Complejidad promedio:
• Búsqueda: O(log n)
• Inserción: O(log n)
• Eliminación: O(log n)
📐

Terminología

Conoce los conceptos fundamentales que usarás en toda la plataforma.

Glosario:
• Raíz: Nodo sin padre
• Hoja: Nodo sin hijos
• Altura: Máxima profundidad
• Profundidad: Distancia a la raíz
• Subárbol: Árbol dentro del árbol
🕸️

¿Qué es un Grafo?

Un grafo G = (V, E) consiste en un conjunto de vértices (V) y un conjunto de aristas (E) que conectan pares de vértices. Modela relaciones entre objetos.

Tipos:
• Dirigido (dígrafo): aristas con dirección
• No dirigido: aristas sin dirección
• Ponderado: aristas con peso
🔗

Representaciones

Los grafos pueden representarse de distintas formas según las necesidades del problema.

Formas comunes:
• Lista de adyacencia: O(V+E) espacio
• Matriz de adyacencia: O(V²) espacio
• Lista de aristas: O(E) espacio
🌐

Aplicaciones reales

Los grafos modelan problemas del mundo real en múltiples dominios.

Usos:
• Redes sociales (amigos)
• GPS y rutas (caminos)
• Internet (páginas web)
• Dependencias de software
➕

Inserción

En un BST, el nuevo nodo se compara con la raíz y desciende por el árbol siguiendo la propiedad BST hasta encontrar una posición vacía.

insert(valor): si raíz es null → raíz = Nodo(valor) si valor < nodo → ir izquierda si valor > nodo → ir derecha repetir recursivamente
🔎

Búsqueda

Aprovecha la propiedad BST para descartar mitades del árbol en cada paso, logrando eficiencia O(log n) en árboles balanceados.

search(valor): si nodo es null → false si valor == nodo → true si valor < nodo → buscar izquierda si valor > nodo → buscar derecha
❌

Eliminación

Tres casos según el nodo a eliminar: sin hijos (hoja), un hijo, o dos hijos (se reemplaza con el sucesor inorden).

Caso 1: hoja → eliminar directamente Caso 2: un hijo → enlazar padre con nieto Caso 3: dos hijos → sucesor = mínimo(subárbol derecho) reemplazar valor, eliminar sucesor
1️⃣

Preorden (NLR)

Visita primero la Raíz, luego el subárbol izquierdo, luego el derecho. Útil para copiar o serializar el árbol.

preorden(nodo): visitar(nodo) // N preorden(izquierda) // L preorden(derecha) // R
2️⃣

Inorden (LNR)

Visita primero izquierda, luego raíz, luego derecha. En un BST produce los valores ordenados ascendentemente.

inorden(nodo): inorden(izquierda) // L visitar(nodo) // N inorden(derecha) // R
3️⃣

Postorden (LRN)

Visita izquierda, derecha y finalmente la raíz. Útil para eliminar el árbol o evaluar expresiones.

postorden(nodo): postorden(izquierda) // L postorden(derecha) // R visitar(nodo) // N
4️⃣

Por Niveles (BFS)

Recorre el árbol nivel por nivel usando una cola. Garantiza visitar todos los nodos de profundidad N antes de pasar a N+1.

niveles(raíz): cola = [raíz] mientras cola no vacía: nodo = cola.dequeue() visitar(nodo) encolar hijos
